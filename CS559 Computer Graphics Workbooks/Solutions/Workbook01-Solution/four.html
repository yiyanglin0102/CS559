Exercise 1: Sliders

In this box, make 3 sliders. Add event handlers such that the third slider shows the difference in value between the first two. So that slider3 = slider2 - slider1. Slider3 should move whenever slider1 or slider2 are moved. Making it such that slider1 and/or slider2 move when slider3 moves is a little trickier (and optional).

Hint: Box 4 on page three is a good starting point. The lower of the two sliders there has a nicer behavior for this exercise. Consider setting the min and max properties of the sliders so that the third slider has the right range. The min can be negative.




Exercise 2: Start Stop

In this box, make a slider and two buttons. One labeled "start" the other labeled "stop". When you press "start", the slider starts moving. When you press "stop" the slider stops. This is sortof like the last slider in Box 6 on page three, except that you should use two buttons, rather than one checkbox.

Hint: With buttons, the easier thing is to handle the onclick events. The event handlers will need to pass information to the animation loop. You'll have to store the state information somewhere.




Exercise 3: Throb, not blink

The "blinking" in Box 7 of page three is really annoying. Instead let's make something a little less annoying: instead of just changing to another color in a flash, have the color slowly (but not too slowly) change from white to red, then back to white. (it should repeat).

Hint: A trick here is that HTML like colors as hex (so red is #FF0000). You'll probably want to do your math with regular numbers and convert. here is a reference page that might be useful.

If you want to try something more challenging, have it fade gradually between a list of colors (just like page 3 switched between red, yellow, and green, you could make yours slowly fade from red to yellow to green then back to red).
